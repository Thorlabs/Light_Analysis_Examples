This is the description of the Thorlabs PM101 remote interface. This single channel industrial Power meter can be remotely controlled using the SCPI (Standard Commands for Programmable Instruments) interface.

In 1975, the IEEE standardized a bus developed by Hewlett-Packard originally called HPIB (Hewlett-Packard Interface Bus), later changed to GPIB (General Purpose Interface Bus). The standard was called IEEE 488 (IEEE 488.1) and it defined mechanical aspects of the bus. The later IEEE 488.2 defined its protocol properties. What was missing were set of rules between the manufacturers on commands to control the instruments. Sometimes these varied even between different models from the same manufacturer.

In 1990 SCPI Consortium released the first SCPI standard as an additional layer for the IEEE-488.2 standard.

SCPI commands are ASCII strings, which are sent to instrument over the physical communication layer. They can perform:

  • Set operations, for example the *RST command (resetting the instrument).
  • Query operations, for example the *IDN? query (querying the instrument’s identification string).

Some SCPI commands exist as both set commands and query commands. An example is an Powermeter's averaging command SENS:AVER. You can set it with the SCPI command SENS:AVER 2, and also query its current value with the SENS:AVER?.

The format mentioned in this manual e.g.: SENSe#:AVERage[:COUNt] <pres> is called canonical form. Here are the most important rules to remember:

  • The parts within square brackets are not mandatory and can be left out.
  • All # within the command represent a suffix positive integer number starting at 1. If there is only one suffix you may skip it because default is applied internally. If there are two or more suffix it is mandatory to specify all.
  • The capital letter parts are mandatory; the small letters can be omitted. This is called short form. An example of the above command in the short form is SENS1:AVER 3. You can use either the short form, or the long form SENSE1:AVERAGE 3, but nothing in between, e.g. SENS1:AVERA 3.
  • The SCPI commands are case-insensitive. You can also use the short form sens1:aver 3 or or the long form sense1:average 3
  • Combine multiple commands into one string using selmicolon ';'. For example, a combined string of SENS1:AVER 3 and SENS1:CORR:WAV 850 is SENS1:AVER 3;CORR:WAV 850. Notice that the second command does not have the SENS1: part. The reason is, that the command tree path does not change within one string. If you want to reset the command tree path to the root, use the colon character at the beginning of the second command: SENS1:AVER 3;:SENS2:CORR:WAV 850.
  • Create query forms by adding a question mark, mostly to the end: SENS1:AVER? Sometimes there is an additional parameter placed after the question mark. There must be a space character between the question mark and the additional parameter. For example: SENS1:CORR:WAV? MIN

The complete SCPI standard is available here: SCPI-99

Measurement Systems

The PM101 supports two different general measurement modes. The following section provides a detail description of these modes. Not all modes are supported by all sensors or operating modes. The Powermeter samples the discrete input signal at 10 kHz. Every millisecond the device processes the last 10 sample results. The device always generates a constant measurement result stream regardless of input signal.

Slow Measurement System

The "Slow Measurement System" is available for all sensor types and measurement modes. It returns values up to a frequency of 1kHz. The command SENS:AVER can be used to scale down the update rate by formula MeasFreq = 1000/AVG. Averaging is considered for the following units only:

  • POW: Photodiode or Thermopile: Sensor power in W or dBm (See command SENSe#:POWer[:DC]:UNIT)
  • CURR: Photodiode: Sensor current in Ampere
  • VOLT: Thermopile: Sensor voltage in Volt
  • PDEN: Photodiode or Thermopile: Sensor power density in W/cm²

All other units have their own independent update rate.

The SCPI commands for this measurement system are part of the MEAS group. This group contains some convenience commands. These commands internally call a predefined SCPI command sequence. Starting with the well-known MEAS? convenience command:

  1. ABOR
    Aborts any ongoing measurement
  2. CONF
    Configures measurement channel for power measurement
  3. INIT
    Starts a new measurement
  4. FETC?
    Blocks until measurement is complete and returns the result

If the same unit should be read multiple times in a row, reconfiguring the channel again and again is not necessary. Therefore, use the READ? convenience command instead:

  1. ABOR
    Aborts any ongoing measurement
  2. INIT
    Starts a new measurement
  3. FETC?
    Blocks until measurement is complete and returns the result

The following diagram shows two examples of SCPI command sequences for the Slow Measure System. There is no timing constraint between the single SCPI commands. Timeline is from left to right.

The first example shows a 1 kHz measurement scenario with an averaging of 1. INIT starts a new measurement. Once the measurement is complete, the result is stored internally until FETC? is called. If FETC? is called multiple times in a row the same result will be returned as no new measurement has been started.

The second example show a 500 Hz measurement scenario with an averaging of 2. INIT starts the new measurement. Before the measurement result is ready FETC? is already called. As the measurement is still ongoing in the background the FETC? blocks the entire remote interface until measurement has been taken.

Once you send a FETC?, MEAS? or READ? command, the interface is blocked until the measurement has been taken. For photodiode sensors these commands work reliable except for very high averaging. High averaging values result in a long block time when waiting for the measurement to terminate. Within this block time do not send an ABOR or any other SCPI command. Please do not enlarge the communication interface timeout to solve this problem but continue reading.

Once a new measurement has been started by INIT, checking for completion can be done by the command FETC:STAT? If this command returns 1, FETC? can be called with providing an instant result. If the FETC:STAT? returns 0, continue to ask for completion or abort the measurement if not longer relevant by using the ABOR command. Do not use the convenience function in these cases.

Another configuration of the Slow Measurement system is the continuous measurement. For this mode there are no convenience functions. Use the following sequence for continuous mode:

  1. ABOR
    Aborts any previous ongoing measurement
  2. CONF:POW
    Configure a power measurement for example
  3. INIT:CONT?
    Start continuous measurement
  4. FETC?
    Returns the last recent measurement result
  5. Delay()
    Delay between reading results
  6. FETC?
    Returns the last recent measurement result
  7. Delay()
    Delay between reading results
  8. ...

Keep in mind in continuous configuration there is no synchronization between the command and the measurements. If FETC? is called faster than the measurement update rate, the same result is read multiple times.

Array Mode

In "Array Mode" the PM101 stores a measurement sequence in the device memory. The mode is supported for photodiode sensors only. The buffered measurements may be accessed at any speed or order once the sequence has been taken. The sequence is always triggered by software. A hardware trigger is not available. The buffer always stores 10k samples with a fixed sample rate of 10 kHz. The buffer contains the raw samples without any further averaging. Data is processed and averaged during data access later. Because of this limitation the product of the parameters delta_t and sample needs to be less or equal 10000 in any case. To get the highest resolution use 10000 samples and delta_t 100. As the sample rate is 10 kHz delta_t needs to be a multiple of 100. Otherwiese delta_t gets coerced.

Before starting a Array Mode for optical power reading ensure:

  • Set manual range (Disables auto ranging of PM101 automatically), command: SENSe#:POWer[:DC]:RANGe[:UPPer]
  • Set full bandwidth for photodiode sensors (no analog filter), command: INPut#[:PDIode]:FILTer[:LPASs][:STATe] 0

The Array Mode is part of the MEAS SCPI command group. You can use the convenience commands like MEAS:ARR? or the following SCPI command sequence to have more control about the measurement:

  1. ABOR
    Abort any ongoing measurement
  2. CONF:ARR 10000, 100
    Configure array mode for 10000 samples with delta_t of 100 us
  3. INIT
    Starts a array measurement
  4. FETC:STAT?
    Scope buffer filled? If not keep on asking. For timeout send ABOR finally
  5. FETC:STAT?
    Scope buffer filled? If not keep on asking. For timeout send ABOR finally
  6. ...
  7. FETC:STAT?
    Scope buffer filled returned true.
  8. FETC? 0
    Read scope buffer content at index 0
  9. FETC? 1
    Read scope buffer content at index 1
  10. ...
  11. FETC? 9999
    Read scope buffer content at index 9999

The buffered measurements might be accessed in a textual manor using FETC? command or in a binary fession using the FETC:ARR? command. If you are using the faster binary FETC:ARR? command you will always receive up to 40 measurements at once. You can access the array buffer in any order. SCPI Registers

The SCPI system defines a status reporting system based on status register sets. Every state or event is mapped to a single bit in the values of the register sets. The PM101 defines the following register sets:

  • AUXilary: Auxiliary system monitoring like shutter or fan.
  • OPERation: Measurement state monitoring.
  • QUESTionalbe: Measure result reliability status monitoring.
  • STDandard Byte: SCPI global system status byte.

Every register set contains 5 registers.

  • CONDition: The Condition register monitors the actual state. For example if a sensor is currently connected or not.
  • PTRansition: The Positive Transition register masks bit changes from 0 to 1 of the Condition register to the Event register.
  • NTRansition: The Negative Transition register masks bit changes from 1 to 0 of the Condition register to the Event register.
  • EVENT: The Event register shows changes of the Condition register. Some conditions only last for a short moment and are otherwise hard to detect.
  • ENABle: The Enable register masks if changes of the Event registers (TODO JS registers plural?) are forwarded to a higher level hierarchy Condition register.

The following image visualizes the logical relation between the registers.

The Condition register always reflects the current state. For example if a sensor is connected or not. The Positive Transition register masks if a certain condition becomes true (bit changes from 0 to 1) to set the bit at the same position in the Event register. For example to set the Event register bit to 1 if the sensor got connected.
The Negative Transition register tests if condition becomes false to update the Event register. For example to set event register bit to 1 if the sensor got disconnected. The Enable register masks if changes of the Event register should be reported to the higher level hierarchy Condition register. The register sets are logically chained. The following section will provide further details about the chaining.

All registers except the STB register group are 16 bit wide. The following paragraph gives a short description of bit mask in general. Bit mask use the binary representation of numbers to store boolean states (true, false) or (active, inactive). This is a memory saving way to store multiple conditions at once. If you look on a decimal number 9 for example the binary representation is 1001 = 1 * 2 ^ 4 + 1 * 2 ^ 0. This bit mask means condition at bit position 1 and 4 are true. All other conditions are false as all other bits are set to 0.
To test if a certain bit is set to 1 you can either use bit logic operations or use generic mathematic operations. Bit logic operations are available in almost every programming language. In Python you would write something like condition = bitmask & (1 << position).
If you want to use standard math you can use the following formula: condition = round_down(bitmask / (2 ^ bitPos)) modulus 2.

The register sets are linked together in an hirarchical way. This enables you to check for a summarized condition only. If the highlevel condition becomes true you can than go down the hierarchy to query the details causing the event.

The STAT:OPER status register set contains state information about the instrument’s normal operation. The register set might report to the SCPI Standard Byte register bit 7 if STAT:OPER:ENAB bit mask is set active.

The QUEStionable status register set contains bits which give an indication of the quality of various aspects of the measurement signal. The register set might report to the SCPI Standard Byte register bit 3 if STAT:OPER:ENAB bit mask is set active.

The AUXilary register set contains state informatino about the instrument auxiliary like the NTC or power supply. The register set might report to the SCPI Standard Byte register bit 0 if STAT:OPER:ENAB bit mask is set active.

STAT:AUX?Queries the SCPI Auxiliary Event register
STATus:AUXiliary[:EVENt]?
Response
  • bitmaskuint 16 bit wide event register bit mask.
Use this command to query the SCPI 16 bit wide Auxiliary Event register bit mask. The Auxiliary register set summarizes the operating state of auxiliary equipment connected to the Powermeter. Reading this event register automatically resets the register back to 0. The register content is a bit mask with a 1 bit for every event occurred since the register has been read the last time. If the same event occurs multiple times the event bit stays 1. To get the meaning of the single bits read the STAT:AUX:COND? description. The event register can store a change of the Auxiliary Condition register. The two transition registers are used to mask changes of the condition register to the event register. If a certain bit in the condition register is set from 0 to 1 and the bit at the same position in the positive transition register STAT:AUX:PTR is set 1, the event register at the bit position is set to 1. It is also possible to detect condition register changes from 1 back to 0 using the negative transition register STAT:AUX:NTR following the same logic.
Changes of the event register can be summarized to the SCPI status register bit 0 if enabled by STAT:AUX:ENAB. To read the SCPI status register call STB?. After reboot or CLS the events are all reset to 0.
Read SCPI register set section to get closer infos about the SCPI register system in general.
